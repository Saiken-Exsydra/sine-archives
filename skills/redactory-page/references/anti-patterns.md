# Anti-Patterns

Do not use these patterns unless the user explicitly asks to break project style.

- SaaS hero/features/pricing/FAQ composition.
- startup gradients, bright blue/purple hero blobs, or tech-company glass cards.
- crypto dashboards, trading screens, or neon hacker UI.
- generic card grids where an archive registry, dossier rail, or editorial spread would fit better.
- oversized rounded cards nested inside other cards.
- decorative charts or metrics unrelated to lore.
- generic icon sets replacing project objects or symbols.
- external stock imagery.
- unreadable faint text on dark backgrounds.
- motion that bounces, wiggles, pops, or feels cartoon-like.
- route/layout rewrites for a local visual task.
- new dependencies for visual polish.
