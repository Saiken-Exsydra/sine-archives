# Writing And UI Rules

## Voice

Use archive-system language:

- Registry
- Dossier
- File
- Addendum
- Restricted
- Observation
- Index
- Seal
- Transmission
- Fragment

Avoid product language:

- Features
- Solutions
- Pricing
- Unlock
- Get started
- AI-powered
- Dashboard

## Readability

- Use clear section titles.
- Keep paragraph measure constrained.
- Provide enough spacing between dense lore blocks.
- Use tables only when comparison or file metadata benefits.
- Use warnings and annotations sparingly.

## Canon Safety

- Do not invent new facts unless the user explicitly asks for expansion.
- If content is missing or ambiguous, preserve the ambiguity instead of filling it with generic lore.
- Prefer citations, cross-links, and existing archive references over new exposition.
