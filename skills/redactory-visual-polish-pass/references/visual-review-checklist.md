# Visual Review Checklist

Ask these questions before patching.

## Identity

- Does the page feel like SiNE Archives or like a generic web template?
- Does the first viewport show the actual system/place/object/entry?
- Are project assets used meaningfully?

## Color And Contrast

- Are backgrounds near black enough?
- Are panels too gray, washed out, or low contrast?
- Are crimson/gold accents controlled?
- Is text readable without losing atmosphere?

## Hierarchy

- Is the main title clear?
- Are metadata, labels, and secondary actions visually subordinate?
- Are sections divided by rules, spacing, or panes?
- Is long prose comfortable to scan?

## Interaction

- Are hover/focus/active states visible?
- Are object interactions discoverable?
- Does motion feel restrained?
- Is reduced motion handled for new animations?

## Common Fixes

- darken overlay opacity.
- reduce card radius.
- add thin ruled separators.
- improve metadata label spacing.
- tighten or loosen section rhythm.
- add subtle crimson/gold accent line.
- replace generic card grid with registry/dossier structure.
